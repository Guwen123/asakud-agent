# mcp-server-config

## 适用场景

- 用户询问如何接入、配置或排查新的 MCP Server。
- 需要区分本地 `stdio` 与远程 HTTP / `streamable-http` MCP Server 的配置字段。
- 需要给出可复制的 `mcpServers` JSON 配置示例。
- 需要基于 MCP 官方文档或可信来源核对最新传输方式、认证字段或兼容性说明。

## 输入

- 用户描述的 MCP Server 类型、运行方式、URL、命令、包名、认证方式或客户端环境。
- 若用户未说明传输方式，默认同时给出本地 `stdio` 与远程 `streamable-http` 两类配置。
- 若用户要求最新口径，必须使用 `fetch_web` 检索 MCP 官方文档或服务提供方文档。

## 工作流程

1. 判断用户要接入的是本地 MCP Server 还是远程 MCP Server；不明确时同时覆盖两种。
2. 检索 MCP 官方文档、registry 或服务提供方文档，核对当前推荐传输方式和字段命名。
3. 汇总通用字段：服务名、传输类型、命令或 URL、认证信息、环境变量。
4. 对本地 `stdio` 配置，说明 `command`、`args`、`env` 的用途，并给出 JSON 示例。
5. 对远程 HTTP 配置，说明 `type` / `transport`、`url`、`headers` / `auth` 的用途，并给出 JSON 示例。
6. 标注旧版或兼容字段，例如 SSE/HTTP+SSE，只在相关时说明不作为默认推荐。
7. 给出一句话配置口诀或检查清单，帮助用户快速落地。

## 可用工具

- `fetch_web`: 检索 MCP 官方规范、registry、客户端配置文档和服务提供方接入说明。

## 输出格式

- 先给出简短结论。
- 按“通用字段 / 本地 stdio / 远程 HTTP / 注意事项”组织答案。
- 提供最小可用 JSON 示例。
- 若信息来自网页检索，保留来源线索或说明已按官方口径核对。
- 若用户缺少关键信息，不要停止；给出两类默认配置模板并提示用户替换占位符。

## 验证

- 检查是否明确区分本地与远程 MCP Server。
- 检查 JSON 示例是否可复制、字段层级是否包含 `mcpServers`。
- 检查密钥、token 等敏感信息是否通过 `env` 或 `headers` 占位符表达，避免写真实凭证。
- 检查是否没有把已弃用或兼容传输方式误写成唯一推荐方案。
