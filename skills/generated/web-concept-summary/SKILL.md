# web-concept-summary

## 适用场景

- 用户要求“搜索一下/查一下/联网看看”某个概念、产品、技术、公司、人物或术语是什么。
- 用户希望得到很短的解释，例如“三句话总结”“一句话说明”“用几句话概括”。
- 适合需要实时网页信息、官方文档或权威来源支撑的简洁问答。

## 输入

- `context["user_input"]`: 用户的当前检索与总结请求，通常包含主题和期望句数。
- `context["recent_summary"]`: 可选的近期上下文，用于理解省略的主题。
- `context["tools"]`: 当前脚本允许调用的工具列表，必须包含 `fetch_web`。

## 工作流程

1. 从用户输入中识别待查询主题，例如产品名、技术名、公司名或术语。
2. 识别输出长度要求；若未指定，默认用三句话总结。
3. 构造简洁搜索词，优先加入“官方”“文档”“overview”“是什么”等能定位权威解释的关键词。
4. 使用 `context["run_tool"]("fetch_web", {"query": query})` 执行网页检索。
5. 基于检索结果提炼定义、核心用途、典型场景或差异点，避免编造未检索到的信息。
6. 按用户要求的句数输出中文总结，并保留必要来源线索。

## 可用工具

- `fetch_web`: 用于搜索网页、打开页面、获取公开信息并辅助事实核验。

## 输出格式

- 直接给出指定句数的中文总结。
- 每句话只表达一个核心信息点。
- 如有来源链接或来源名称，应自然附在对应句子后。
- 如果搜索结果不足或来源不明确，应明确说明不确定性。

## 验证

- 检查是否实际调用了 `fetch_web`。
- 检查总结句数是否符合用户要求。
- 检查定义、用途、对比关系等关键信息是否能由检索结果支持。
- 检查没有把搜索结果之外的推断包装成事实。
