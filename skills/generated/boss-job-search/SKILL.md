# boss-job-search

## 适用场景

- 用户要求查询 Boss 直聘上的岗位、实习、校招、远程岗位或特定技术方向岗位。
- 适合“查 Java 实习”“查上海后端实习”“查远程 Python 岗位”“查 200/天以上 Java 实习”等招聘检索任务。
- 当用户明确指定 Boss 直聘、Boss、BOSS 直聘，或已保存该技能用于岗位检索时，优先使用本技能。

## 输入

- `context["user_input"]`: 用户的岗位查询需求，可能包含关键词、城市、岗位类型、薪资、经验、学历、远程/现场、数量等条件。
- `context["recent_summary"]`: 可选的近期上下文，用于补全连续对话中的城市、岗位方向或筛选偏好。
- `context["tools"]`: 当前脚本可用工具列表，必须包含 `fetch_web`。

## 工作流程

1. 解析用户输入，提取岗位关键词、城市、岗位类型、薪资门槛、经验/学历要求、远程要求和返回数量。
2. 构造面向 Boss 直聘岗位页面或搜索结果的检索语句，包含“Boss 直聘”、岗位关键词、城市和岗位类型。
3. 调用 `fetch_web` 获取公开网页结果；如果结果过宽或不足，使用更精确的查询词再次检索。
4. 从检索结果中整理岗位名称、公司、城市、薪资、经验/学历要求、发布时间或更新信息、链接和来源线索。
5. 优先展示与用户条件最匹配的前 10 条左右结果；如用户指定数量，则按指定数量输出。
6. 对无法确认的信息标注“未显示”或“需打开页面确认”，避免编造岗位细节。
7. 如果 Boss 直聘结果不足，可说明检索限制，并给出可继续细化的条件建议。

## 可用工具

- `fetch_web`: 用于搜索公开网页、打开页面、提取岗位信息和来源线索。

## 输出格式

- 先用一句话说明检索条件和结果数量。
- 使用 Markdown 表格输出岗位列表，建议列为：岗位、公司、城市、薪资、要求、发布时间、链接/来源。
- 表格后补充筛选说明、信息缺口和可继续细化的条件。
- 如果检索失败或结果不足，直接说明原因，不生成虚假岗位。

## 验证

- 检查输出岗位是否与关键词和地点匹配。
- 检查薪资、城市、公司、链接等信息来自检索结果。
- 检查未检索到的信息没有被补写或推断为事实。
- 检查脚本返回包含 `{"output": "..."}`。
