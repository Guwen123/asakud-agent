# Query Rules

- Prefer concise search queries built from the user's key entity, location, and intent.
- For job-search tasks, include platform, role, city, internship/full-time, and core tech keywords.
- For stock or market tasks, include ticker/name, market, date words such as today/latest, and the target metric.
- If the first result is too broad, run a second focused query rather than inventing details.
- Keep source uncertainty in the output when fetched content is incomplete.
