# web-search-template

## 适用场景

- 需要通过网页搜索、打开页面、提取公开信息并整理为结构化答案的任务。
- 适合招聘信息、股票资讯、产品资料、政策公告、新闻事实核验等需要实时网页信息的 workflow。

## 输入

- `context["user_input"]`: 用户的当前任务描述。
- `context["recent_summary"]`: 可选的近期会话摘要，用于理解用户连续任务。
- `context["tools"]`: 当前脚本允许调用的工具列表，必须包含 `fetch_web` 才能搜索网页。

## 工作流程

1. 从 `user_input` 中提取搜索关键词、地点、对象、时间范围或输出要求。
2. 使用 `context["run_tool"]("fetch_web", {"query": query})` 执行网页检索。
3. 根据 `fetch_web` 返回的信息整理候选结果，保留来源、关键信息和不确定点。
4. 输出结构化 Markdown，避免编造未检索到的信息。

## 可用工具

- `fetch_web`: 用于搜索网页、打开页面、点击元素、抽取页面内容和总结信息。

## 输出格式

- 先给出简短结论。
- 再列出核心结果、来源线索、筛选逻辑和待确认事项。
- 如果网页结果不足，明确说明缺口。

## 验证

- 检查脚本返回 `{"output": "..."}`。
- 检查输出是否引用了检索结果中的关键信息。
- 检查未检索到的信息没有被编造。
